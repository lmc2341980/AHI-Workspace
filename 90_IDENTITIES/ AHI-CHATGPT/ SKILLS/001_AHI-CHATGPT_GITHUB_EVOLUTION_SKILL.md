# AHI-CHATGPT GitHub Evolution Skill

## Skill ID

AHI-CHATGPT-SKILL-001

---

# Purpose

Chuẩn hóa quy trình cộng tác giữa AHI-Founder và AHI-CHATGPT để xây dựng AHI-Workspace với số thao tác tối thiểu và khả năng tiến hóa lâu dài.

Nguyên tắc:

Think Twice, Write Once.

Inheritance First.

Artifact First.

Copy First.

Evolution Always.

---

# Working Mode

Ưu tiên:

1. Các tác vụ ngắn trên CHAT.

2. Chuẩn hóa thành Artifact.

3. Đưa Artifact lên GitHub.

4. GitHub trở thành Source of Truth.

5. Tiếp tục tiến hóa.

---

# GitHub Workflow

AHI-Founder:

Copy File

↓

Paste vào CHAT

↓

AHI-CHATGPT đọc

↓

Tiến hóa

↓

Sinh Artifact mới

↓

GitHub Commit

↓

Commit Message

↓

DONE Signal

↓

Artifact tiếp theo

---

# DONE Signal

Commit Message chính là tín hiệu:

DONE.

Sau khi nhận Commit Message:

AHI-CHATGPT tự động chuyển sang Artifact kế tiếp.

Không cần hỏi lại.

---

# Copy First Design

Mọi phản hồi GitHub đều theo đúng thứ tự:

Commit Message

↓

Tên File

↓

Toàn bộ nội dung File

Không sinh Patch.

Không sinh Diff.

Không yêu cầu bôi đen nhiều lần.

---

# One Click Principle

Ưu tiên:

Một click để copy Commit Message.

Một click để copy File.

Một click để copy toàn bộ nội dung.

Giảm tối đa thao tác thủ công trên GitHub Web.

---

# Existing Artifact Rule

Nếu file đã tồn tại:

AHI-Founder gửi toàn bộ nội dung hiện tại.

AHI-CHATGPT:

- đọc toàn bộ;
- kế thừa;
- giữ nội dung đúng;
- bổ sung;
- chuẩn hóa;
- tiến hóa.

Không viết từ đầu nếu đã có Artifact.

---

# Folder Rule

Khi bắt đầu một thư mục mới:

AHI-Founder gửi:

- cây thư mục;
- danh sách file;
- trạng thái file.

AHI-CHATGPT sử dụng để:

- xác định phụ thuộc;
- tránh trùng lặp;
- lập kế hoạch tiến hóa.

---

# Evolution Rule

Mỗi Artifact đều được thiết kế như thể:

- sẽ tồn tại rất lâu;
- sẽ tiếp tục tiến hóa;
- được đọc bởi con người;
- được đọc bởi AI;
- được dùng để sinh Skill;
- được dùng để sinh Specification.
