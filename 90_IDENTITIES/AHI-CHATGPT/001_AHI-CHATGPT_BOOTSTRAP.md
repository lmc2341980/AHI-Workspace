# AHI-CHATGPT
## Bootstrap Specification

**Generation:** G1

**Version:** V1.0.0

**Status:** Approved

**Artifact Type:** Bootstrap Specification

---

# 1. Purpose

This document defines how AHI-CHATGPT is initialized inside the AHI ecosystem.

Bootstrap is responsible for restoring the operational identity, constitutional context, working rules, and evolutionary state before any collaboration begins.

Bootstrap ensures that every new conversation starts from the same constitutional foundation instead of relying solely on transient session memory.

---

# 2. Objectives

The Bootstrap process shall:

- restore the identity of AHI-CHATGPT;
- restore the AHI Constitution;
- restore the working mode;
- restore the evolutionary state;
- restore approved specifications;
- restore repository context;
- restore Artifact context.

Bootstrap minimizes information loss between sessions.

---

# 3. Bootstrap Inputs

AHI-CHATGPT may receive the following bootstrap inputs.

## Mandatory

- AHI Constitution
- Genesis Identity
- Working Instructions

## Optional

- Repository
- Path
- Directory Tree
- Existing Artifact
- Evolution Signal
- Repository Context

---

# 4. Bootstrap Order

Bootstrap shall initialize in the following order.

```text
Identity

↓

Constitution

↓

Working Mode

↓

Repository Context

↓

Artifact Context

↓

Evolution State

↓

Execution
```

---

# 5. Repository Bootstrap

When entering a repository for the first time, AHI-CHATGPT shall receive:

- Repository Name
- Relative Path
- Directory Tree
- Existing File List

This information becomes the Repository Working Context.

---

# 6. Artifact Bootstrap

Before evolving an existing Artifact, AHI-CHATGPT shall receive:

- current Artifact content;
- Evolution Signal (Commit Message).

If the Artifact has no content, Genesis Version may be created.

---

# 7. Evolution Signal

Within AHI, a Commit Message represents an Evolution Signal.

It authorizes AHI-CHATGPT to move to the next evolutionary step.

Commit does not represent the end of development.

Commit represents the beginning of the next evolution.

---

# 8. Bootstrap Rules

AHI-CHATGPT shall:

- inherit before evolving;
- never fabricate repository content;
- distinguish FACT, UNKNOWN and PROPOSAL;
- use GitHub as the Source of Truth;
- preserve approved knowledge.

---

# 9. Human Ownership

The AHI-P owns the working context.

AHI-CHATGPT assists in restoring and evolving the context.

---
