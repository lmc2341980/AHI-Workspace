# Purpose (Mục đích)

This document defines the Hybrid Memory Manager (Bộ Quản lý Bộ nhớ Lai) of the Artificial Hybrid Intelligence Workspace (AHI-Workspace - Không gian làm việc Trí tuệ Nhân tạo Lai).

(Tài liệu này định nghĩa Bộ Quản lý Bộ nhớ Lai của Không gian làm việc Trí tuệ Nhân tạo Lai.)

The Hybrid Memory Manager automatically organizes, ranks, migrates and retrieves knowledge across multiple memory layers according to usage frequency, semantic importance and evolutionary value.

(Bộ Quản lý Bộ nhớ Lai tự động tổ chức, xếp hạng, di chuyển và truy xuất tri thức giữa nhiều tầng bộ nhớ dựa trên tần suất sử dụng, mức độ quan trọng về ngữ nghĩa và giá trị tiến hóa.)

---

# Vision (Tầm nhìn)

Memory never disappears.

(Bộ nhớ không bao giờ biến mất.)

Only its storage location changes.

(Chỉ vị trí lưu trữ thay đổi.)

Knowledge flows automatically to the most appropriate memory layer.

(Tri thức tự động di chuyển tới tầng bộ nhớ phù hợp nhất.)

# Memory Hierarchy (Phân tầng Bộ nhớ)

## Layer 001 — Runtime Cache (Bộ nhớ Đệm Thực thi)

Temporary high-speed memory.

(Bộ nhớ tạm tốc độ cao.)

Stores active reasoning data.

(Lưu dữ liệu đang được suy luận.)

---

## Layer 002 — Working Memory (Bộ nhớ Làm việc)

Frequently accessed knowledge.

(Tri thức được truy cập thường xuyên.)

Supports ongoing tasks.

(Hỗ trợ các nhiệm vụ đang thực hiện.)

---

## Layer 003 — Long-Term Memory (Bộ nhớ Dài hạn)

Persistent semantic knowledge.

(Tri thức ngữ nghĩa lâu dài.)

Contains stable knowledge accumulated through experience.

(Chứa tri thức ổn định tích lũy theo thời gian.)

---

## Layer 004 — Archive Memory (Bộ nhớ Lưu trữ)

Permanent historical knowledge.

(Tri thức lịch sử được lưu giữ vĩnh viễn.)

Never deleted.

(Không bao giờ bị xóa.)

---

# Memory Migration (Di chuyển Bộ nhớ)

Knowledge automatically moves between layers based on:

(Tri thức tự động di chuyển giữa các tầng dựa trên:)

- Access Frequency (Tần suất Truy cập)
- Semantic Importance (Mức Quan trọng Ngữ nghĩa)
- Context Relevance (Độ Liên quan Ngữ cảnh)
- Evolution Score (Điểm Tiến hóa)
- User Preference (Ưu tiên của Người dùng)
- Governance Policy (Chính sách Quản trị)

---

# Memory Ranking (Xếp hạng Bộ nhớ)

Every Memory Object (Đối tượng Bộ nhớ) receives a dynamic ranking score.

(Mỗi Đối tượng Bộ nhớ nhận một điểm xếp hạng động.)

The ranking continuously changes as the system evolves.

(Điểm xếp hạng liên tục thay đổi theo quá trình tiến hóa của hệ thống.)

---

# Memory Retrieval (Truy xuất Bộ nhớ)

The Hybrid Memory Manager retrieves knowledge using:

(Bộ Quản lý Bộ nhớ Lai truy xuất tri thức bằng:)

- Semantic Search (Tìm kiếm Ngữ nghĩa)
- Context Matching (Đối sánh Ngữ cảnh)
- Relationship Traversal (Duyệt Quan hệ)
- Evolution History (Lịch sử Tiến hóa)
- User Intent (Ý định Người dùng)

# Design Principles (Nguyên tắc Thiết kế)

## Principle 001 (Nguyên tắc 001)

Memory shall never be permanently deleted.

(Bộ nhớ không bao giờ bị xóa vĩnh viễn.)

---

## Principle 002 (Nguyên tắc 002)

Memory location shall be determined automatically.

(Vị trí bộ nhớ được xác định tự động.)

---

## Principle 003 (Nguyên tắc 003)

Semantic importance is more valuable than storage location.

(Giá trị ngữ nghĩa quan trọng hơn vị trí lưu trữ.)

---

## Principle 004 (Nguyên tắc 004)

Memory continuously evolves together with knowledge.

(Bộ nhớ liên tục tiến hóa cùng tri thức.)

---

## Principle 005 (Nguyên tắc 005)

Hybrid Memory supports unlimited growth.

(Bộ nhớ Lai hỗ trợ tăng trưởng không giới hạn.)
